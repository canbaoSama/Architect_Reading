# MCP 协议与 Agent Skills

> 参考资料：阿里云、Agent Skills 工程化实践、MCP 官方文档

## 四层工具栈的演化

| 层级 | 概念 | 作用 |
|------|------|------|
| **Prompt** | 人机接口 | System prompt（角色/风格）+ User prompt（具体任务） |
| **Agent** | 自主执行 | 从问答升级到可拆分任务、调用工具、多轮迭代 |
| **Agent Skills** | 可复用工作流 | 将经验、流程、规则沉淀成标准化能力包 |
| **MCP** | 协议层 | 让 AI 安全、标准化地访问外部工具和资源 |

## MCP（Model Context Protocol）

### 核心功能

- **连接协议**：统一 AI 与外部系统的接入方式
- **通信方式**：基于 JSON-RPC 2.0 标准
- **解决的问题**：「能不能做」—— 外部系统能否被 AI 调用

### 典型应用

- 文件系统、数据库、API 接入
- 浏览器控制、终端操作
- 第三方服务（天气、翻译、搜索等）

## Agent Skills 与 MCP 的分工

| 维度 | MCP | Agent Skills |
|------|-----|-------------|
| **功能** | 连接协议 / 工具接入 | 流程固化 / 工作标准 |
| **解决的问题** | 「能不能做」 | 「怎么稳定地做对」 |
| **形式** | 协议 + 工具实现 | SKILL.md + config + 脚本 |

**协同模式**：MCP 提供工具，Skills 定义使用规则；一个 Skill 可编排多个 MCP 服务。

## Agent Skills 标准目录结构

```
skill-name/
├── SKILL.md        # 自描述指令 + 工作流步骤
├── config.yaml     # 硬约束与参数
├── scripts/        # 确定性脚本（Python/Shell）
└── references/     # 模板、规范、示例
```

### 核心特点

1. **流程固化**：把组织流程、最佳实践固化为可复用包
2. **渐进披露**：只在需要时加载技能内容
3. **解决痛点**：Prompt 难复用、流程不统一、输出不稳定

## 工程化落地路径

1. **定义 Skill 边界**：单一职责，复杂任务拆分为独立技能
2. **标准化描述**：用 JSON Schema 而非纯自然语言，降低模型幻觉
3. **接入 MCP**：将 Skills 注册到 MCP 服务，处理并发、重试、错误熔断
4. **版本与安全治理**：像微服务一样管理技能迭代和供应链安全

## 设计原则：预分类与优先级

当没有相应工具时，调用工具智能体无法返回大模型本身就能回答的信息。很多内容可能：

- 已在会话历史中回答
- 在文档中有相关信息
- 无需调用工具即可完成

**建议**：先做预分类，处理历史记录、文档检索，仅在必要时再调用工具智能体。可参考阿里百炼、百度千帆、腾讯云 TI、华为 ModelArts 等平台。
