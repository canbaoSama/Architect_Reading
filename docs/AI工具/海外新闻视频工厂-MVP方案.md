# 海外新闻视频工厂 - MVP方案

## 1. 文档用途

本文件用于指导开发智能体或工程团队实现“海外新闻视频工厂”的最小可用版本（MVP）。

MVP 目标不是一步到位，而是验证以下问题：

1. 能否稳定抓取海外新闻并生成候选
2. 能否把单条新闻做成一条可发布的 35 秒短视频
3. 视频质量是否达标
4. 后期通过自然语言控制是否顺手
5. 成本与效率是否满足预期

本文件对应的是验证版方案；完整长期版请见《海外新闻视频工厂-完整方案.md》。

---

## 2. MVP目标

### 2.1 业务目标

先只验证单一主路径：

> 一条海外新闻 -> 一条 35 秒中文快讯视频

### 2.2 用户体验目标

运营者通过聊天入口完成以下动作：

- 查看今日候选新闻
- 选择一条新闻制作视频
- 调整脚本风格
- 触发渲染
- 获取成片和发布文案

### 2.3 质量目标

MVP 的视频应满足：

1. 有明确开头钩子
2. 字幕与旁白同步
3. 来源可见
4. 风格统一
5. 画面可看，不廉价
6. 你本人愿意实际发布测试

---

## 3. MVP边界

### 3.1 本阶段必须做

1. 3-5 个海外新闻源
2. 新闻抓取与正文抽取
3. 单条候选新闻制作
4. 中文快讯脚本生成
5. 图片与网页截图素材
6. TTS 配音
7. 字幕时间轴
8. 一套高质量快讯模板
9. 聊天式文字控制
10. 成片回传

### 3.2 本阶段不做

1. 多平台自动发布
2. 多模板体系
3. 大规模视频素材抓取
4. 完整风险审查系统
5. 完整复盘与看板
6. 多账号管理
7. 深度 QClaw 编排
8. 大规模素材资产库
9. 自动化全链路无人值守发布

---

## 4. MVP推荐架构

```text
你（Telegram / 飞书 / 简单聊天入口）
        │
        ▼
   [命令解释器]
        │
        ▼
   [FastAPI 后端]
        │
 ┌──────┼──────┬──────┐
 ▼      ▼      ▼      ▼
抓新闻  写脚本  抓图/截图  TTS+字幕
        │
        ▼
    Remotion 渲染
        │
        ▼
   回传成片 + 标题建议
```

说明：

- MVP 可以先不接 QClaw，把聊天入口做成 Telegram Bot 或飞书 Bot
- 架构上保留未来接入 QClaw 的可能

---

## 5. MVP功能路径

## 5.1 候选新闻获取

系统定时抓取少量稳定新闻源，输出候选列表。

### 用户行为

用户发送：

- 今天候选
- 给我今天 10 条海外科技新闻

### 系统输出

每条候选包括：

- 标题
- 来源
- 发布时间
- 一句话摘要
- 是否建议制作

---

## 5.2 选择新闻并生成脚本

### 用户行为

例如：

- 做第 2 条
- 做第 2 条，35 秒，偏快讯

### 系统输出

- 中文脚本
- 3 个标题
- 2 个封面字
- 1 个评论区提问

---

## 5.3 调整脚本

### 用户行为

- 开头更炸一点
- 别太像新闻联播
- 更口语一点
- 加一句这对普通人意味着什么

### 系统输出

- 新版本脚本
- 更新标题建议

---

## 5.4 生成视频

### 用户行为

- 开始渲染

### 系统流程

1. 拉取首图和网页截图
2. 生成配音
3. 生成时间轴
4. 渲染 35 秒视频
5. 回传结果

---

## 5.5 成片回传

### 输出

- mp4
- 预览图
- 标题建议
- 简介建议
- 评论区引导

### 用户下一步

- 通过
- 重写开头
- 换图
- 重渲染

---

## 6. MVP模块拆分

## 6.1 数据层

### 最少需要的表

1. news_sources
2. news_items
3. video_jobs
4. scripts
5. job_assets
6. audio_outputs
7. video_outputs
8. command_logs

### 核心要求

- 能追踪每个任务做到哪一步
- 能记录失败原因
- 能保留历史脚本版本

---

## 6.2 新闻采集模块

### MVP信源建议

只接 3-5 个：

- Reuters
- BBC
- TechCrunch
- The Verge
- 一个官方 press release 源（可选）

### 功能

- RSS 或 feed 拉取
- 标题/链接/发布时间保存
- URL 去重
- 入库

### 要求

必须支持手动触发抓取，方便测试。

---

## 6.3 正文抽取与截图模块

### 功能

- 抽正文
- 抽首图
- 截网页首屏图
- 标准化内容结构

### 技术建议

- trafilatura
- readability
- Playwright 截图

### 输出字段

- cleaned_content
- hero_image
- page_screenshot
- source
- published_at

---

## 6.4 候选排序模块

### MVP实现方式

先用轻规则，不用做复杂智能排序。

### 简单评分规则

- 是否有正文
- 是否有首图
- 是否有网页截图
- 是否来自白名单信源
- 是否够新

### 输出

分三类即可：

- 可做
- 一般
- 不建议

---

## 6.5 脚本生成模块

### 目标

把新闻转成 35 秒快讯脚本。

### 输出结构建议

```json
{
  "hook": "...",
  "body": ["...", "...", "..."],
  "ending": "...",
  "titles": ["...", "...", "..."],
  "cover_texts": ["...", "..."],
  "comment_prompt": "..."
}
```

### Prompt 要求

至少维护 3 套 prompt：

1. 初稿 prompt
2. 口语化 prompt
3. 爆点增强 prompt

### 功能要求

支持基于上一版脚本重写。

---

## 6.6 素材模块

### MVP素材策略

只使用这四类：

1. 原文首图
2. 原文网页截图
3. 官网/产品图
4. logo / 自制信息卡

### 功能

- 下载图片
- 保存本地或对象存储
- 统一命名
- 生成适合模板的比例版本

### 不做

- 大规模抓取视频素材
- 大规模版权判断

---

## 6.7 TTS 与时间轴模块

### 功能

- 生成固定音色旁白
- 输出音频文件
- 生成句级字幕时间轴

### 技术建议

- 选择一套稳定 TTS
- 优先使用 alignment 数据
- 回退 Whisper

### MVP要求

先只支持一种主播声音，不做多音色。

---

## 6.8 视频模板模块

### MVP只做一个模板：

# 35 秒海外快讯模板

### 结构建议

1. 0-3 秒：强钩子
2. 3-8 秒：来源确认
3. 8-18 秒：事件主画面
4. 18-28 秒：背景补充
5. 28-35 秒：影响与提问

### 技术建议

- Remotion

### 模板要求

1. 深色背景
2. 大字排版
3. 关键词高亮
4. 来源标签固定
5. 动效克制但精致

---

## 6.9 聊天入口模块

### MVP建议

先使用：

- Telegram Bot
或
- 飞书 Bot

而不是一开始就强依赖 QClaw。

### 原因

MVP 阶段优先验证视频生产闭环，而不是复杂前台。

### 最少支持命令

- 今天候选
- 做第 N 条
- 开头更炸一点
- 更口语一点
- 开始渲染
- 给我标题
- 给我简介

---

## 6.10 审核回传模块

### 功能

回传：

- 成片
- 标题建议
- 简介建议
- 评论引导

### 支持动作

- 通过
- 重写开头
- 换图
- 重渲染

---

## 7. MVP详细开发任务清单

以下任务按建议顺序排列。

## Phase 0：项目初始化

### 任务

1. 初始化 Git 仓库
2. 创建基础目录结构
3. 确定 Python / Node 版本
4. 配置 `.env.example`
5. 搭 Docker 基础环境
6. 建立 README 和运行说明

### 验收

- 项目可本地启动
- 配置管理清晰

---

## Phase 1：数据库与任务状态

### 任务

1. 设计表结构
2. 建 migration
3. 实现 ORM
4. 定义 job status 枚举
5. 实现基础 CRUD

### 验收

- 可以创建 video_job
- 可以记录任务状态和错误

---

## Phase 2：新闻抓取

### 任务

1. 接入 3-5 个信源
2. 写 RSS 抓取器
3. URL 去重
4. 标题相似度去重
5. 入库 news_items
6. 增加手动抓取接口
7. 增加定时任务

### 验收

- 能稳定抓到候选新闻
- 重复新闻不会重复入库

---

## Phase 3：正文抽取与截图

### 任务

1. 实现正文抽取器
2. 实现首图抽取
3. 实现网页截图
4. 写入结构化字段
5. 增加失败回退

### 验收

- 选一篇新闻，能拿到正文、首图、截图

---

## Phase 4：候选新闻输出

### 任务

1. 实现候选列表接口
2. 实现简单评分规则
3. 实现消息格式化
4. 支持“今天候选”查询

### 验收

- 用户能在聊天工具中看到候选新闻列表

---

## Phase 5：脚本生成

### 任务

1. 设计脚本 JSON 结构
2. 编写初稿 prompt
3. 编写口语化 prompt
4. 编写爆点增强 prompt
5. 生成标题、封面字、评论引导
6. 存储脚本版本
7. 支持脚本重写

### 验收

- 选一条新闻后可自动生成中文快讯脚本
- 支持至少两轮重写

---

## Phase 6：素材抓取与标准化

### 任务

1. 下载首图
2. 保存网页截图
3. 下载 logo/官网图（如可行）
4. 素材统一命名
5. 生成模板适配比例
6. 写入素材 metadata

### 验收

- 每条 job 至少能生成 2-3 个可用素材

---

## Phase 7：TTS 与时间轴

### 任务

1. 接入 TTS
2. 固定一套主播音色
3. 实现语速参数
4. 输出音频文件
5. 获取 alignment 或跑 Whisper
6. 生成字幕时间轴 JSON

### 验收

- 一条脚本能生成可用旁白和字幕时间轴

---

## Phase 8：Remotion 模板

### 任务

1. 设计模板视觉规范
2. 实现 35 秒快讯模板
3. 接入图片/截图/字幕/音频
4. 实现基础动效
5. 支持预览渲染
6. 支持正式导出
7. 解决字体加载问题

### 验收

- 输入一条脚本、几张图、一个音频，可以输出一条完整视频

---

## Phase 9：聊天入口

### 任务

1. 建 Telegram/飞书 Bot
2. 接收文本命令
3. 做简单命令解析
4. 路由到后端 API
5. 返回候选列表、脚本、视频链接

### 验收

- 用户能通过聊天完成一次完整制作流程

---

## Phase 10：审核与输出

### 任务

1. 回传成片
2. 回传标题/简介建议
3. 支持通过 / 重写 / 重渲染
4. 归档最终视频产物

### 验收

- 运营者能完成“看稿 -> 调整 -> 出片 -> 通过”的完整闭环

---

## 8. MVP验收标准

MVP 完成后，必须满足以下标准才算合格：

### 功能验收

1. 能抓到候选新闻
2. 能抽出正文
3. 能拿到至少一种可用图像素材
4. 能生成中文脚本
5. 能生成配音
6. 能生成字幕时间轴
7. 能渲染出 35 秒视频
8. 能通过聊天命令驱动整个流程

### 质量验收

1. 视频不是廉价感
2. 来源清晰
3. 配音自然
4. 字幕同步
5. 模板统一
6. 你本人愿意拿去发平台测试

---

## 9. MVP推荐技术栈

### 后端

- Python
- FastAPI
- PostgreSQL（或先 SQLite，推荐直接 PostgreSQL）
- Redis

### 抓取

- requests / httpx
- Playwright
- Trafilatura

### AI

- 1 个脚本模型
- 1 个 TTS 提供商
- Whisper / alignment

### 视频

- Remotion
- FFmpeg

### 入口

- Telegram Bot 或 飞书 Bot

---

## 10. MVP工作量级别

建议按有效开发工作量评估。

MVP 大致量级：

- 40 - 70 有效开发小时

粗略分配：

- 项目骨架与数据层：6 - 10h
- 新闻抓取与正文抽取：8 - 12h
- 脚本生成与 prompt：6 - 10h
- 素材抓取与标准化：5 - 8h
- TTS 与时间轴：4 - 8h
- Remotion 模板：8 - 14h
- Bot 入口与回传：4 - 8h
- 联调与修复：5 - 10h

---

## 11. MVP完成后的下一步

MVP 验证通过后，再进入完整方案的增强阶段：

1. 多模板
2. 更强素材源
3. 更稳的状态管理
4. QClaw 深度接入
5. 复盘和推荐系统

---

## 12. 最终建议

MVP 的核心不是“做很多”，而是：

> 做出一条你愿意发布的、质量合格的海外新闻短视频。

因此优先级必须是：

1. 模板质量
2. 脚本质量
3. 配音质量
4. 素材逻辑
5. 聊天操作顺滑度

如果这 5 点都达标，MVP 就成功了。
