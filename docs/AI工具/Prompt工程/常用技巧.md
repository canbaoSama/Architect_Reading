# Prompt 常用技巧

从基础到进阶的 Prompt 优化思路，适合日常开发与调试。

## 1. 基础优化

### 1.1 明确角色定义

❌ 「帮我写登录功能」

✅ 「作为前端工程师，使用 Vue3 + TypeScript + Element Plus，实现一个带验证码、记住密码、错误提示的登录表单」

### 1.2 结构化输出

❌ 「给我一些优化建议」

✅ 「按以下格式输出：1. 问题描述 2. 影响范围 3. 建议方案 4. 优先级（高/中/低）」

### 1.3 提供上下文

包含：项目背景、当前问题、性能指标、目标用户等，便于模型给出更贴合的回答。

## 2. 中级技巧

| 技巧 | 说明 |
|------|------|
| **Chain of Thought** | 先分析、再给出结论，减少跳跃与幻觉 |
| **Few-Shot** | 提供 1–3 个示例，约定格式和风格 |
| **约束条件** | 明确技术栈、格式、字段、长度限制 |
| **对比分析** | 用表格对比多种方案的优劣 |
| **错误预演** | 预先列出可能错误与应对策略 |

## 3. 高级技巧

- **元提示（Meta-Prompting）**：先让 AI 分析需求，再优化 Prompt
- **动态角色切换**：模拟多角色视角讨论问题
- **渐进式优化**：分轮次从浅到深迭代
- **多维度评估**：从技术、业务、用户等维度评估

## 4. 实用模板

### 代码 review

```
角色：资深前端 Code Reviewer
任务：对以下代码进行 review
输出格式：
1. 潜在问题（按严重程度：高/中/低）
2. 优化建议
3. 安全与性能注意点
```

### Bug 排查

```
角色：有经验的调试工程师
上下文：项目技术栈、报错信息、复现步骤
任务：分析可能原因，给出排查步骤和修复方案
要求：先列出假设，再逐条验证
```

### 需求拆解

```
角色：产品经理 + 技术架构师
任务：将以下需求拆解为可开发的任务列表
输出：每个任务包含 描述、依赖、预估工时、验收标准
```
