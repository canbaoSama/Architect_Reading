## 技术难点攻克
1. 如何解决cesium中的材质问题：通过学习部分 glsl 着色器知识，搭配AI自己仿照开发。好像有一个叫Shader Graph 的可视化编程，以节点连接的方式构建复杂着色器逻辑。
2. 如何实现cesium的各类特效：自己写方法，通过turf调取一些特殊计算方法，然后自己修改填充。里面很多参数全是a、b、c、d,很难识别
3. 如何实现cesium各图形的平滑过渡：先是直接更新，会出现闪烁、改为callbakcProperty方法返回最新值，每分钟自己切16次数据片更新，但是还是会出现跳动、改为跟随requestAnimateFrame方法切片数据，效果提升了但是还是不佳，最后找到Cartesian3.lerp()方法，通过格林时间间隔去更新数据，完美。
4. 如何设计AI智能体的处理逻辑以及对每个部分的接入：当没有相应工具处理时，调用工具智能体是没法返回大模型本身就能返回的信息，并且很多内容可能已经在之前的会话历史中回答过了，或者在文档中有相关信息，那么就更没必要调用工具智能体了，所以还是先设计了预分类操作，处理历史记录、文档信息等。如果返回需要调用工具，再去调用工具智能体。其实可以参考阿里云的百炼控制台、百度智能云千帆大模型平台、腾讯云 TI 平台、华为云 ModelArts
