# 缓存与存储常用技巧

## 1. 新标签页进入会展示广告，后面无论刷新还是内部跳转都不会出现广告

考察的是 localStorage、sessionStorage 和 cookie 的作用域：

- **SessionStorage**：存储当前会话数据，仅在当前标签页和窗口有效，包括当前会话内的页面跳转；新开页面无法共享；关闭标签页后清除。适合「仅首次展示」场景。
- **LocalStorage**：永久存储直到显式删除，同一源下所有页面可访问。
- **Cookie**：可设置过期时间；未设置时视为会话 Cookie，关闭浏览器窗口后删除。

```js
document.addEventListener('DOMContentLoaded', () => {
    const adFlag = sessionStorage.getItem('advertisementShown')
    if (adFlag) {
        document.querySelectorAll('.ad-class').forEach(ad => (ad.style.display = 'none'))
    } else {
        sessionStorage.setItem('advertisementShown', 'true')
    }
})
```

## 2. 后端传送的图片 URL 列表，如何获取数量、统计出错、保证渲染时间一致？

- **数量获取**：统计渲染容器下的子节点数量。
- **统计出错数量**：`onload` / `onError` 监听，或全局监听按标签名判断。
- **渲染时间一致**：
  - 禁用缓存：URL 加 `?t=${Date.now()}`
  - 预加载：`Promise.all` 预加载后再渲染
  - fetch：`{ cache: 'no-store' }`
  - meta 标签：`Cache-Control: no-cache, no-store, must-revalidate`
