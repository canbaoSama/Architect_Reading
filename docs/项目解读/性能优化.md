# 性能优化案例

## vue3的增量水合，只有当组件窗口被打开时才会加载
* 传统水合的弊端：在传统的 Vue 应用中，即使是服务器渲染 (SSR) 的应用，客户端也需要一次性下载并水合整个应用。 这会导致：
    * 首屏渲染阻塞： 在水合完成之前，用户无法与页面进行交互。
    * 资源浪费： 即使某些组件在初始视图中不可见，它们仍然会被水合。

* 增量水合的优势：增量水合允许 Vue 逐步水合组件，而不是一次性完成。 这样做的好处是：
    * 更快的可交互时间 (TTI)： 用户可以更快地与页面进行交互，因为不需要等待整个应用水合完成。
    * 更好的性能： 只有可见或即将可见的组件才会被水合，减少了不必要的资源消耗。
### 如何实现
1. 利用<Suspense>，用于处理异步依赖。 它可以包裹任何异步组件，并在异步组件加载完成之前显示一个占位内容
2. defineAsyncComponent 是 Vue 3 提供的一个函数，用于定义异步组件。 它可以接受一个返回 Promise 的函数，该 Promise resolve 的结果应该是一个组件定义。

## 资源加载优化和懒加载
1. 预先压缩、构建代码时压缩；采用webP 格式，压缩效率更高。
2. 图片懒加载，只有当滚动到窗口视图是才去加载。现代浏览器原生支持 loading="lazy" 属性，可以轻松实现图片懒加载。
    ```html
    <img src="image.jpg" loading="lazy" alt="My Image">
    ```
3. 图片预加载，通过Promise.all方法预先加载下个界面需要的图片，但是很少遇到这样的情况。
4. 构建时设置assetsInlineLimit，将小图片直接转为base64编码，不再需要http请求。
5. CDN缓存，例如七牛云可以自动帮忙切图片大小
6. 雪碧图

## echarts 性能优化
1. 部分动画采用css实现。
2. 数据抽样： 对数据进行抽样，减少数据量。 每隔 N 个点取一个点，或者使用平均值、最大值、最小值等统计指标代替原始数据。
3. 开启 progressive 属性： 启用 progressive 属性，ECharts 会将数据分成多个批次进行渲染，避免页面卡顿。
4. 使用 appendData 方法： 使用 appendData 方法向图表中追加数据，ECharts 会自动进行增量更新，只渲染发生变化的部分
