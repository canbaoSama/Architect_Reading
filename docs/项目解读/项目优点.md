## 图片编辑器
1. 比对 Fabric.js、PixiJS，还是选择了leaferjs这款轻量级的2D图形渲染引擎，学习成本低、性能更高、轻量、社区活跃度够、issue解决速度不错、而且支持跨平台，支持扩展。
2. 我们将操作效率放在第一位，根据标记设计每一个功能和交互细节，提供的常用操作比华为的多，
3. 因为我们的选框内部会有一些固定长款的提示组件，这些组件方向、大小等都不是简单的跟随选框变化就行了，所以遇到的放大、缩小、旋转这几个功能的计算方式都需要额外编写。
    这些组件为什么不单独跟选框分来呢，单独计算呢，因为要跟鼠标交互，这个时候判断是否空元素或者选中元素就比较复杂，所以综合考虑还是放在一个box中。
4. Leaferjs 和 PixiJS: 这两个框架通常被认为是性能最强的。 它们都直接利用 WebGL 或 Canvas API 进行底层渲染，避免了额外的抽象层，因此在处理大量图形对象或复杂动画时，通常表现出色。 Leaferjs 在某些特定场景下可能会略胜一筹，因为它在框架设计上更加注重性能优化，例如采用了更高效的渲染管线和更精细的内存管理

## cesium 3D 仿真
我这个项目基本上是实现了战场仿真需要的所有态势，优势如下
1. 各类图形的绘制：基础图形包括点、线、面、圆、多边形、长方形、直线箭头、攻击箭头、夹击箭头等
                特殊图形包括辅助侦查矩形、辅助侦查锥形、导弹抛物线、雷达扫描等等。
2. 特殊材料绘制：一些场景需要一些流动效果，比如水波纹材质、流动线条材质、发光线条材质。
3. 特殊场景开发：一些场景的开发，比如阳光照射、雨、雪、雾等，全球水淹模拟效果等。
4. glsl编写：GLSL (OpenGL Shading Language) 是一种用于编写着色器（shaders）的编程语言。 着色器是在图形处理单元（GPU）上运行的小程序，用于控制图形渲染管线的各个阶段
5. 做了一个仿真播放器，支持前进、后退、倍速度。
