# 事件循环与异步

> 来源整合：`知识点/浏览器/事件循环机制.md`

## 一、是什么

`JavaScript` 在设计之初便是单线程，同一时间只能做一件事。为解决单线程运行阻塞问题，使用**事件循环（Event Loop）**机制。

### 任务分类

- **同步任务**：立即执行，直接进入主线程
- **异步任务**：进入任务队列，等待主线程空闲时再执行

流程：同步任务 → 主执行栈 → 异步任务 → 任务队列 → Event Loop 轮询

## 二、宏任务与微任务

### 微任务

在主函数执行结束之后、当前宏任务结束之前执行。

- `Promise.then`
- `MutationObserver`
- `process.nextTick`（Node.js）

### 宏任务

- `script`（外层同步代码）
- `setTimeout` / `setInterval`
- UI 渲染 / UI 事件
- `postMessage`、`MessageChannel`
- `setImmediate`、I/O（Node.js）

### 执行顺序

1. 执行一个宏任务
2. 遇到微任务则加入微任务队列
3. 当前宏任务完成后，依次执行微任务队列
4. 再执行下一个宏任务

## 三、async 与 await

- `async` 函数返回 `Promise`
- `await` 会阻塞后面代码，将其加入微任务队列
- `await` 后跟非 Promise 则直接返回对应值

## 四、面试回答要点

1. 同步任务入栈执行，异步任务交 WebAPIs 处理后回调进入 Callback Queue
2. 执行栈为空时，Event Loop 从队列取任务入栈
3. Event Loop 由宿主环境（如浏览器）实现
4. WebAPIs 由 C++ 实现的浏览器线程处理
