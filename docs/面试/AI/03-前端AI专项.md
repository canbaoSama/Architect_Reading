## 一、SSE 与 WebSocket 对比

| 维度 | SSE | WebSocket |
|------|-----|-----------|
| **通信方向** | 服务端 → 客户端（单向） | 双向全双工 |
| **协议** | 基于 HTTP | 独立 ws/wss |
| **数据格式** | 文本 | 文本 + 二进制 |
| **重连** | 浏览器自带自动重连 | 需自行实现心跳和重连 |
| **实现** | EventSource 即可 | 服务端需支持协议升级 |
| **适用场景** | 通知、日志流、**AI 流式输出** | 聊天、协同编辑、游戏、直播 |

**简要记忆**：
- SSE：客户端连上后，服务端持续推送；适合 AI 流式回复
- WebSocket：双方随时互发；适合实时双向交互

**前端工具**：`@microsoft/fetch-event-source`（SSE）、`react-use-websocket`（WebSocket）

## 二、AI 核心概念（必背）

| 概念 | 说明 |
|------|------|
| **RAG** | 先检索外部知识，再让模型回答；解决幻觉、知识陈旧 |
| **Agent** | 可自主规划、调用工具、逐步完成任务的智能体 |
| **FunctionCall** | 模型调用外部接口的标准方式（查天气、查库等） |
| **MCP** | 模型与外部系统/工具通信的统一协议 |
| **Skills** | 可复用的能力封装（写代码、总结文档等） |

**关系**：用户需求 → Agent 作为中枢 → 用 RAG 查资料、FunctionCall 调工具、Skills 执行能力 → 通信依托 MCP。

## 五、IDE 对比（面试常问）

| IDE | 特点 |
|-----|------|
| VS Code | 生态丰富、轻量，日常开发主力 |
| WebStorm | 智能提示、重构强，适合大型项目 |
| Cursor | AI 原生，代码生成与对话一体，AI 开发首选 |
| Zed | 启动快、操作流畅，偏效率 |

**推荐说法**：日常用 VS Code，AI 开发用 Cursor，大型项目用 WebStorm。

## 六、AI 在业务中的应用场景

- 智能客服、内部问答：RAG + 对话界面
- 代码生成与补全：Cursor、Copilot 等
- 需求/接口文档自动生成与总结
- 前端页面自动生成（描述/草图 → 代码）
- 自然语言转数据可视化、报表
- 测试用例、测试脚本自动生成

---

## 七、需求如何用 AI 实现（拆解模板）

1. **需求拆解**：大需求 → 小模块，明确每块职责
2. **AI 规划 /plan**：目标、步骤、输入输出、依赖、时间
3. **方案设计 /spec**：接口、数据结构、页面逻辑、提示词、异常处理
4. **实现**：前端界面 + 模型调用 + RAG / FunctionCall
5. **测试**：功能、幻觉、流式稳定性、异常
6. **上线与迭代**：根据效果优化 prompt、流程和参数

## 八、Agent 底层原理

### ReAct
- **推理 + 行动**：思考 → 选工具/动作 → 执行 → 观察 → 再思考
- 多数智能体的核心逻辑

### Transformer
- 大模型基础架构
- 自注意力机制，理解上下文和语义
- 前端侧只需理解：用于理解语言和生成内容
