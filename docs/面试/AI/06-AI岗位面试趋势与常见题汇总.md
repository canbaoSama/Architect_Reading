# AI 岗位面试趋势与常见题汇总

> 面向 **2025–2026** 招聘市场的整理：市场更看重**可验证的工程判断、上线经验与系统设计**，而非单纯背诵八股。

## 一、现在 AI 面试「真正在筛什么」

### 1. 岗位分化更明显（准备要对准 JD）

| 方向 | 面试侧重（概括） |
|------|------------------|
| **AI / LLM 应用工程师** | RAG、Agent、工具调用、评测与观测、延迟/成本、Prompt 与结构化输出 |
| **ML 工程师（偏传统 + 生产）** | 指标与验证、特征与数据、训练/服务、漂移与回滚、ML 系统设计 |
| **研究 / 训练侧** | 论文级理解、实验设计、训练/推理 infra、安全与对齐（视公司） |
| **数据科学家** | SQL、因果/实验、经典 ML；GenAI 占比因团队而异 |

同一 title 在不同公司（**产品型** / **infra 型** / **模型厂**）考察深度差很多，**先读 JD 与业务类型**，再分配复习权重。

### 2. 题型权重在偏移（和几年前不同）

- **ML / AI 系统设计**：端到端链路、约束下的取舍（延迟、成本、数据新鲜度、评测、监控）——许多流程里这是**区分 senior 的主战场**。
- **生产与排障**：传统场景类似「线上指标掉了如何拆因」；LLM 场景则延伸到检索质量、幻觉、prompt/模型版本变更等。
- **编码**：部分 senior 岗 **DSA 减少或变短**；更常见**实用 Python**、小功能实现、**读代码/改 bug**；大厂通用 SWE 通道仍可能考中等 LeetCode。
- **项目深挖**：真实 trade-off、失败案例、**可核对数字**（数据规模、延迟、成本、业务指标）——部分 recruiter 在初筛就会问「带数字的经历」。
- **新形态**（逐渐出现）：允许 AI 辅助的一轮 + 禁止 AI 的一轮；代码评审；take-home 中评估「在 AI 生成基础上你多做了什么」。核心是**可追问的理解**，而非默写。

### 3. checklist 里高频出现的「硬技能块」

1. **工程底座**：Python、API、Git、测试、可维护的模块划分。  
2. **ML 基础（能讲清行为即可）**：过拟合、验证方式、指标选择、误差分析。  
3. **LLM 应用**：Prompt、JSON/结构化输出、function calling、失败与降级。  
4. **RAG**：分块、embedding、检索（稠密/稀疏/混合）、rerank、引用与可追溯。  
5. **评测与 LLMOps**：离线/在线指标、回归集、tracing、版本对比、成本与延迟监控。  
6. **部署与可靠性**：容器、配置、限流、缓存、批处理、故障模式。  
7. **Agent**：规划、工具循环（如 ReAct）、权限与安全边界（越权、注入）。  
8. **行为面**：ownership、冲突、失败复盘——用于区分「只会背题」与「能落地」。

## 二、常见面试题汇总（强化版：基础 + 进阶 + 场景）

### A. 机器学习基础

**基础题**
- 偏差–方差、过拟合怎么发现与缓解？  
- 分类/回归常用指标？不平衡数据怎么办？  
- 交叉验证怎么做、什么时候会泄漏？  
- L1/L2 正则各自倾向什么解？  
- 树模型 vs 线性模型适用场景？  
- 离线指标好但线上差，可能原因？

**进阶题**
- AUC、PR-AUC、F1 在业务里如何选？为什么同一个模型在两套指标下排序不同？  
- 样本偏移（sample selection bias）与标签噪声（label noise）怎么区分？  
- 特征重要性不稳定时，你如何判断模型是否可解释、可上线？  
- 如何设计一个误差分析框架，快速定位“哪类样本最容易错”？

### B. ML 系统设计与生产

**基础题**
- 设计一个推荐/搜索/风控模型的**端到端**系统（数据、训练、上线、监控）。  
- 训练–服务不一致如何缓解？特征新鲜度如何保证？  
- 如何做 A/B？如何定义核心指标与护栏指标？  
- 数据漂移、概念漂移如何监控？触发重训的策略？  
- **场景题**：「精度一夜之间从 91% 掉到 76%，第一小时你怎么查？」（模型坏 / 数据变 / 标注流程变 / 上游特征 bug）

**进阶题**
- 多模型并存（Champion/Challenger）时，灰度发布和回滚机制如何设计？  
- 特征平台如何避免“离线算得出、在线算不出”的一致性问题？  
- 在线推理高峰期 GPU/CPU 资源不足，如何做限流、降级、批处理？  
- 模型版本更新后，如何证明“收益来自模型变化而不是流量变化”？

### C. Transformer / LLM 基础（视岗位深度）

**基础题**
- Attention 在算什么？复杂度？KV cache 解决什么问题？  
- 预训练 vs 指令微调 vs RLHF（各解决什么问题）？  
- 上下文长度受限时如何组织信息？「lost in the middle」指什么、怎么缓解？  
- 幻觉常见原因与缓解手段（检索、约束、引用、后处理）？

**进阶题**
- temperature、top-p、top-k 分别影响什么，线上如何调参更稳？  
- 为什么长上下文不一定更好？你如何做“少而精”的上下文构建？  
- Function calling 与纯文本输出相比，可靠性和维护成本差异在哪里？  
- 模型升级（如 4o -> 新版本）后，兼容性风险点有哪些？

### D. RAG（极高频）

**基础题**
- RAG 端到端流程（索引管线 + 在线检索生成）口述或画图。  
- RAG vs 微调：何时用哪个、能否组合？  
- 稠密检索 vs 稀疏（BM25）vs 混合；何时要 rerank？  
- 分块策略有哪些、各有什么代价？  
- 如何评测 RAG（检索：Recall@k、MRR；生成：忠实度、相关性；延迟与成本）？  
- 生产上：索引更新、过期文档、权限与多租户、失败降级怎么做？  
- 进阶：Adaptive / Agentic / Corrective RAG 各自解决什么问题（一句话 + 一例即可）？

**进阶题**
- 文档更新频繁时，如何做增量索引，避免全量重建？  
- 多语言、多格式文档（PDF、网页、表格）如何统一清洗和切分？  
- 召回率高但答案质量低，应该先改检索还是先改生成？如何判断？  
- 引用命中率高但用户满意度低，可能问题在哪一层？

**场景题**
- 设计“公司制度问答”系统：要求强引用、低幻觉、权限隔离。  
- 设计“售后客服助手”：要求低延迟、可追踪、支持工单回写。  
- 文档质量很差（目录乱、重复多）时，你如何先做数据治理再上 RAG？

### E. Prompt、工具与安全

**基础题**
- 如何写可维护的 prompt（版本、变量、少样本）？  
- 结构化输出怎么保证可解析？  
- Function calling / tool use 设计时要注意什么？  
- Prompt injection、敏感数据进模型：风险与防护思路？

**进阶题**
- 如何设计 prompt A/B 实验，避免“主观感觉变好了”？  
- 当工具返回脏数据或超时时，Agent 应如何恢复？  
- 对外部工具调用如何做参数白名单、超时控制、审计日志？

### F. Agent

**基础题**
- LLM 与 Agent 区别？典型循环（如 ReAct）？  
- 何时不该用 Agent（过度复杂、成本、不可控）？  
- 多步任务如何限制步数、预算、工具权限？

**进阶题**
- 单 Agent 与多 Agent 何时切换？如何避免协作开销大于收益？  
- Agent 记忆何时写入、何时清理，如何避免错误记忆污染？  
- 如何评估 Agent：任务完成率、工具成功率、平均步数、成本？

### G. 推理、成本与系统设计（GenAI）

**基础题**
- 如何压延迟（并行检索、缓存、更小模型路由、流式）？  
- 如何压成本（缓存命中、批处理、模型级联、长短请求分流）？  
- 设计一个「企业内部文档问答」或「客服机器人」的完整架构（含观测与评测）。

**进阶题**
- 多模型路由如何设计策略：便宜模型先试，失败再升级？  
- 如何做 token 成本预算，避免月度费用失控？  
- 模型厂商切换时，如何做抽象层避免大面积重构？

### H. 编码（按公司准备）

**基础题**
- 字符串/哈希表/滑动窗口/二叉树等**中等**题仍常见；部分岗加 **PyTorch 小实现**或数据处理。  
- 能口述复杂度、边界条件；能写可测的小函数。

**进阶题**
- 手写一个简化版的 LRU 缓存、限流器、重试器。  
- 给一段 LLM/RAG 代码，找出隐藏 bug（并发、空值、超时、异常吞掉）。  
- 读一段性能差的 Python 代码，说明如何优化（I/O、批处理、向量化）。

### I. 行为与项目

**基础题**
- 印象最深的技术失败？你具体做了什么？  
- 与 PM/业务意见不一致怎么推进？  
- 你如何用数据证明「上线后变好了」？  
- 准备 2–3 个**带数字**的项目故事（规模、指标、取舍）。

**进阶题**
- 项目中你做过最关键的 trade-off 是什么？为什么不是另一个方案？  
- 你如何定义“这次上线成功了”？如果失败，你的止损机制是什么？  
- 如果给你再做一次，你会删掉什么复杂度？

## 四、各岗位对应知识点（完整清单，讲清楚版）

### 1) AI 应用工程师（LLM Engineer）
- Python 与后端基础：会写 API、异步调用、错误处理。  
- LLM 基础：知道 token、上下文、采样参数如何影响结果。  
- RAG：会做切分、索引、检索、重排、引用。  
- Agent 与工具调用：会做函数调用和失败重试。  
- 评测：会定义“答得对不对、快不快、贵不贵”。  
- 工程化：日志、监控、灰度、回滚。

### 2) ML 工程师（MLE）
- 统计与 ML 基础：指标、验证、误差分析。  
- 训练与特征：数据处理、特征工程、训练调参。  
- 部署与服务：在线推理、扩缩容、延迟优化。  
- 监控与迭代：漂移检测、重训策略、版本治理。  
- 工程能力：可测试代码、流水线、自动化。

### 3) 数据科学家（DS）
- SQL 与数据分析：能快速定位业务问题。  
- 统计实验：A/B、显著性、因果推断基础。  
- 建模与解释：模型能解释给业务听。  
- 数据可视化与报告：能把结论讲明白。  
- 与工程协作：把分析结果落地成策略。

### 4) Prompt 工程师
- Prompt 设计：角色、上下文、约束、示例。  
- 结构化输出：JSON schema、字段校验。  
- 评测优化：建立样本集，做 A/B。  
- 安全防护：注入攻击识别与防御。  
- 版本管理：不同 prompt 版本的效果追踪。

### 5) AI 产品经理
- AI 能力边界：知道模型能做什么、不能做什么。  
- 场景拆解：把模糊需求拆成可执行流程。  
- 指标体系：质量、效率、成本三类指标。  
- 风险意识：幻觉、合规、隐私。  
- 跨团队协作：研发、算法、运营协同推进。

### 6) AI 测试/评测工程师（LLM Eval）
- 测试方法：功能、鲁棒性、安全、对抗测试。  
- 评测集建设：覆盖真实场景和边界样本。  
- 自动化回归：版本切换前后自动比对。  
- 质量门禁：设置上线阈值和回退条件。  
- 报告能力：把问题定位到“检索/模型/prompt/工具”。

### 7) AI 平台/LLMOps 工程师
- 模型网关：统一鉴权、路由、限流。  
- 多模型管理：不同模型的切换和降级。  
- 观测体系：调用链、失败率、延迟、成本。  
- 成本治理：缓存、路由、预算告警。  
- 平台化思维：让业务团队低成本复用能力。

### 8) MLOps/平台工程师
- 训练流水线：数据到模型的自动化流程。  
- 模型仓库：版本、元数据、可追踪性。  
- 部署治理：灰度、回滚、弹性扩缩容。  
- 特征平台：离线/在线一致性。  
- 运维能力：告警、故障排查、SLA 管理。

### 9) AI 安全/对齐工程师
- 攻击面理解：注入、越权、数据泄露。  
- 安全策略：输入过滤、输出审查、权限隔离。  
- 安全评测：红队样本、攻击复现。  
- 合规与隐私：脱敏、审计、权限控制。  
- 防护工程化：把策略变成可执行规则。

### 10) 数据工程师（AI 向）
- 数据采集与清洗：去重、规范化、质量校验。  
- 标注与数据集管理：版本、质量、可追踪。  
- 向量与索引供给：为 RAG 提供高质量语料。  
- 数据链路稳定性：任务调度、重试、监控。  
- 成本效率：存储、计算、吞吐优化。

### 11) AI 前端工程师
- 对话交互：多轮状态、流式渲染、中断与重试。  
- 可解释 UI：引用来源、工具调用轨迹展示。  
- 性能优化：长列表、流式卡顿、首屏速度。  
- 多模态体验：文本、图片、文件上传交互。  
- 安全体验：敏感操作确认、权限提示。

### 12) 应用算法工程师（CV/NLP/语音）
- 方向基础：CV/NLP/语音核心方法。  
- 数据与训练：数据增强、损失函数、调参。  
- 任务指标：识别率、召回率、时延等。  
- 部署优化：模型压缩、蒸馏、推理加速。  
- 业务落地：指标和业务效果闭环。

### 13) AI 解决方案架构师
- 行业理解：把业务流程映射为 AI 流程。  
- 技术选型：模型、工具、部署方案。  
- 架构设计：可扩展、可运维、可交付。  
- 沟通能力：面向客户解释技术与价值。  
- 交付管理：PoC 到生产的里程碑推进。

### 14) AI 研究工程师 / 研究员
- 论文阅读与复现：快速抓重点并可验证。  
- 实验设计：变量控制、结果可信。  
- 训练能力：分布式训练、调参策略。  
- 创新能力：提出并验证改进方向。  
- 研究表达：技术报告、分享与答辩。

## 五、高效使用本文档

1. **按目标岗位**从第二节划掉不考的块（例如纯 DS 可能弱化 Agent）。  
2. 每个大块准备：**1 张架构草图 + 3 个追问答案**（为什么不用 B 方案、失败时怎么办）。  
3. **系统设计与线上排障**各练几次计时口述，通常比刷冷门定义回报更高。  
4. 简历与口头叙述对齐：**同一项目里的数字和名词**在面试中保持一致。
