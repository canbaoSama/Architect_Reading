# AI 方向面试准备

> 面向**前端转 AI 应用开发**的面试知识体系，基于 2024-2026 行业趋势整理，涵盖 OpenClaw、LangGraph、Rules/Skills、MCP 等主流技术。

## 先看这里（新手必读）

如果你刚开始看这套资料，第一页看不太懂很正常。建议先按下面顺序读：

1. 先看「术语 5 分钟速读」：搞懂高频词，不被概念卡住
2. 再看「入门阅读顺序」：按 01 → 02 → 03 的顺序建立整体认知
3. 最后做一个小项目：把知识变成可讲的项目经验

目标不是一次全懂，而是先建立一张地图：**LLM 是大脑，RAG 是查资料，Agent 是执行者，前端是用户交互入口**。

## 术语 5 分钟速读

- **LLM**：大语言模型，本质是“根据上下文生成下一个 token”
- **Prompt**：你给模型的指令。提示词写得越清楚，输出越稳定
- **RAG**：先检索业务知识，再让模型回答，减少“胡说八道”
- **Embedding**：把文本变成向量，用于语义检索
- **Agent**：能拆解任务、调用工具、循环执行的智能体
- **FunctionCall / Tool Call**：模型调用外部能力（查天气、查数据库、发请求）
- **MCP**：让模型统一连接外部工具与系统的协议
- **SSE / WebSocket**：前端常用流式通信方式，支撑 AI 打字机式输出体验

## 7 天最小可行入门计划（前端转 AI）

- **Day 1**：读完 01，理解 LLM/RAG 基础概念，能口述 RAG 流程
- **Day 2**：读完 02，练 10 条 Prompt，做 1 个 FunctionCall 小例子
- **Day 3**：读完 03，手写一个支持流式输出的对话页面
- **Day 4**：把“文档问答”做成最小 demo（输入问题 -> 返回答案）
- **Day 5**：加上引用来源、错误处理、重试，完善工程细节
- **Day 6**：复盘并写项目说明（目标、架构、难点、优化）
- **Day 7**：模拟面试：自我介绍 + 项目讲解 + 追问问答

## 学完后你至少要能讲清楚这 4 件事

1. **为什么业务里要上 RAG，而不是直接问模型**
2. **一次用户提问在系统里是怎么流转的（前端 -> 模型/检索 -> 前端）**
3. **前端在 AI 应用中的价值（体验、稳定性、可观测、交互效率）**
4. **你做过的 AI 小项目里，遇到过什么问题、如何优化**

## 面试趋势（2025）

- **GenAI 占比约 75%**：RAG、提示词工程、Agent 设计成为主流考点，传统 ML（CNN、梯度下降等）约占 20-30%
- **前端转 AI 优势**：懂交互 + 懂 AI 的「AI 界面 specialist」需求旺盛
- **典型面试轮次**：4-6 轮，2-4 周，包含系统设计、深挖技术、行为面试
- **必考场景**：设计 RAG 客服聊天机器人、文档问答、多 Agent 协作系统

## 前端转 AI 核心能力要求

| 能力 | 说明 |
|------|------|
| 大模型与提示词 | 了解基本原理，能设计与优化 prompt |
| RAG | 能基于 RAG 搭建私有知识库问答 |
| Agent | 理解工作流程，会使用 FunctionCall |
| **主流框架** | 了解 OpenClaw、LangGraph、Rules/Skills、MCP 等，至少熟悉其一 |
| 前端+AI 项目 | 对话界面、流式输出、实时响应 |
| 方案设计与落地 | 需求拆解、AI 方案设计、上线迭代 |

## 常见岗位方向

- **AI 应用工程师**：调用 LLM API、搭建 RAG、集成到业务
- **前端 AI 工程师**：AI 交互、流式 UI、对话体验
- **AI 产品 / 全栈**：从产品到前端的完整 AI 体验
