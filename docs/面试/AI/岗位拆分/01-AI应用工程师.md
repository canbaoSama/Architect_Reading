# AI 应用工程师（LLM 应用）面试题与知识点

## 1. 岗位定位

- 核心目标：把大模型能力接入真实业务，做成可上线、可维护的应用。  
- 典型工作：RAG、Agent、工具调用、评测回归、线上监控与优化。

## 2. 知识点清单（讲清楚版）

- **Python 与后端基础**：会写 API、异步调用、重试与异常处理。  
- **LLM 基础**：知道上下文窗口、采样参数、幻觉原因。  
- **RAG 全链路**：文档清洗、切分、向量化、检索、重排、引用。  
- **Prompt 与结构化输出**：可维护模板、JSON schema、字段校验。  
- **工具调用**：function calling、超时处理、兜底策略。  
- **评测体系**：准确性、引用命中、延迟、成本、失败率。  
- **上线能力**：日志、监控、灰度、回滚、权限与安全。

## 3. 常见面试题

### 基础题
- 你如何设计一个 RAG 系统的端到端流程？  
- RAG 和微调分别适合什么场景？  
- 如何保证模型输出结构化且可解析？  
- 如何减少幻觉并提高可追溯性？

### 进阶题
- 召回率高但答案质量差，你先改检索还是先改生成？为什么？  
- 多模型路由怎么设计：便宜模型先试，失败再升级可行吗？  
- 文档频繁更新时，如何做增量索引并保证一致性？  
- 如何用一套评测集支持多版本 prompt/模型回归？

### 场景题
- 设计一个企业制度问答助手：要求低幻觉、强引用、权限隔离。  
- 线上突然延迟翻倍，你如何在 30 分钟内排查并止损？  
- 工具服务不稳定时，如何保证主流程仍可用？

## 4. 面试回答要点

- 优先说“约束条件”：时延、成本、准确性、安全。  
- 明确“失败处理”：超时、空召回、低置信度怎么兜底。  
- 给出真实数字：P95 延迟、成功率、成本降幅、命中率变化。

## 5. 面试题参考答案（精选）

### Q1：你如何设计一个 RAG 系统的端到端流程？
**答题框架**：离线索引 + 在线检索生成 + 评测监控。  
**参考答案**：  
1) 离线：文档清洗 -> 语义切分 -> 向量化 -> 建索引，保留文档来源和权限标签。  
2) 在线：用户问题向量化 -> 检索 TopK -> 重排 -> 构建上下文 -> 模型生成。  
3) 输出：返回答案 + 引用片段 + 置信提示。  
4) 监控：记录命中率、回答正确率、P95 延迟、token 成本，并做回归评测。

### Q2：RAG 和微调分别适合什么场景？
**参考答案**：  
- 数据经常变化、需要引用来源、强调事实准确：优先 RAG。  
- 需要稳定风格、固定格式、特定任务行为：可考虑微调。  
- 实际项目常见做法是“RAG 优先，微调补充”，先用 RAG 验证价值，再评估微调 ROI。

### Q3：召回率高但答案质量差，先改检索还是先改生成？
**参考答案**：  
先排查检索。因为“检索错，生成再强也会错”。  
先看检索结果是否相关（TopK 质量、重排效果、切分粒度），再看 prompt 是否正确使用引用；最后再调模型参数。

### Q4：线上延迟翻倍，30 分钟怎么止损？
**参考答案**：  
先止损：降级到轻量模型/减少上下文长度/关闭非核心工具调用。  
再定位：看链路分段耗时（检索、重排、模型、工具）。  
最后修复：针对瓶颈做缓存、并行、限流或回滚到上一版本，并补充告警阈值。
