# ML 工程师（传统 + 生产）面试题与知识点

## 1. 岗位定位

- 核心目标：把机器学习模型稳定地训练、部署、监控并持续优化。  
- 典型工作：特征工程、模型训练、线上推理、漂移监控、重训迭代。

## 2. 知识点清单（讲清楚版）

- **统计与 ML 基础**：指标、验证、偏差方差、误差分析。  
- **数据与特征工程**：清洗、特征构造、特征一致性。  
- **训练与调参**：模型选择、超参、实验管理。  
- **服务化部署**：在线推理、扩缩容、性能优化。  
- **监控与重训**：漂移检测、阈值告警、重训策略。  
- **工程化实践**：流水线、版本管理、灰度与回滚。

## 3. 常见面试题

### 基础题
- 如何判断模型过拟合？怎么缓解？  
- 离线指标与线上指标不一致通常是什么原因？  
- 如何设计一个模型上线流程？  
- 你监控哪些线上指标？

### 进阶题
- 数据漂移和概念漂移如何区分？  
- Champion/Challenger 机制如何设计？  
- 训练服务和推理服务特征不一致怎么办？  
- 高延迟场景如何做模型压缩或推理优化？

### 场景题
- 模型精度一夜下降，你第一小时做什么？  
- 新特征上线后效果不稳定，如何验证与回滚？  
- 线上反馈变慢导致标签延迟，你如何调整评估策略？

## 4. 面试回答要点

- 回答尽量“流程化”：发现问题 -> 定位 -> 实验 -> 上线 -> 监控。  
- 强调可追踪性：数据版本、模型版本、实验记录。  
- 用真实业务指标讲结果：CTR、转化率、坏账率、召回率等。

## 5. 面试题参考答案（精选）

### Q1：如何判断过拟合并缓解？
**参考答案**：  
训练集效果持续变好但验证集变差就是典型信号。缓解手段包括：正则化、早停、特征降噪、增加数据、降低模型复杂度。

### Q2：离线指标好但线上差，常见原因？
**参考答案**：  
常见是数据分布变了、特征口径不一致、标签延迟、线上规则干预。先核对数据链路一致性，再排查流量结构和线上策略。

### Q3：模型精度一夜下降，第一小时做什么？
**参考答案**：  
先确认监控是否误报，再比较输入分布和预测分布，检查最近变更（特征、模型、规则、数据源），必要时立即回滚到上个稳定版本。

### Q4：Champion/Challenger 怎么设计？
**参考答案**：  
主模型全量服务，挑战模型小流量并行打分；达标后逐步提流量。核心是可回滚、可对比、可解释。
