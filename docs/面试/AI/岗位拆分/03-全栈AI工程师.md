# 全栈 AI 工程师面试题与知识点

## 1. 岗位定位

- 核心目标：独立完成 AI 应用从前端到后端、从原型到上线的闭环。  
- 典型工作：接口与服务、RAG/Agent、前端交互、监控与部署。

## 2. 知识点清单（讲清楚版）

- **前后端全栈能力**：前端交互 + 后端 API + 数据库。  
- **AI 应用能力**：RAG、Prompt、工具调用、Agent 编排。  
- **系统设计能力**：缓存、限流、队列、异步任务。  
- **部署运维能力**：容器化、环境配置、日志与告警。  
- **质量保障**：接口测试、回归测试、灰度发布。  
- **成本与性能**：模型路由、并发控制、Token 成本管理。

## 3. 常见面试题

### 基础题
- 如何从 0 到 1 设计一个 AI 问答应用？  
- 前端、后端、模型服务三者如何分层？  
- 会话状态放前端还是后端？如何权衡？  
- 如何处理工具调用失败与请求超时？

### 进阶题
- 高并发下，如何保证稳定性与成本可控？  
- 如何设计缓存策略，避免重复调用模型？  
- 如何把评测回归接入 CI/CD？  
- 模型或供应商切换时，如何降低改造成本？

### 场景题
- 48 小时内交付一个 PoC，到底先做哪些功能？  
- 用户量突然增长 10 倍，哪里最先成为瓶颈？  
- 出现线上事故（高失败率 + 高成本）时如何排障和回滚？

## 4. 面试回答要点

- 用“分层架构 + 关键链路”讲清楚系统。  
- 重点说取舍：快交付 vs 高质量、低成本 vs 高效果。  
- 强调闭环：需求 -> 实现 -> 观测 -> 迭代。

## 5. 面试题参考答案（精选）

### Q1：如何从 0 到 1 设计一个 AI 问答应用？
**参考答案**：  
第一阶段做最小闭环：登录 -> 提问 -> 检索 -> 回答 -> 引用。  
第二阶段做稳定性：重试、超时、降级、日志、告警。  
第三阶段做优化：评测回归、缓存、路由、成本治理。

### Q2：会话状态放前端还是后端？
**参考答案**：  
短期展示状态放前端（输入中、流式中），核心业务状态放后端（消息历史、引用、工具日志）。这样既保证体验，也保证一致性与可追溯。

### Q3：高并发下如何保证稳定性和成本？
**参考答案**：  
入口限流 + 语义缓存 + 模型路由 + 超时降级。优先保证主流程可用，再逐步恢复高质量能力。

### Q4：模型供应商切换时如何降低改造成本？
**参考答案**：  
做 Provider 抽象层，统一请求/响应协议；业务侧只依赖内部接口，不直接耦合某一家模型 SDK。
