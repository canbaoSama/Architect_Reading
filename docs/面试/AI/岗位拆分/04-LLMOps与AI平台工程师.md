# LLMOps / AI 平台工程师面试题与知识点

## 1. 岗位定位

- 核心目标：建设可复用的 AI 平台能力，保障稳定性、可观测性和成本效率。  
- 典型工作：模型网关、路由策略、调用治理、监控告警、预算控制。

## 2. 知识点清单（讲清楚版）

- **网关能力**：统一鉴权、限流、配额、审计。  
- **路由能力**：按任务/成本/质量动态选择模型。  
- **可靠性能力**：重试、熔断、降级、超时控制。  
- **观测能力**：链路追踪、成功率、P95 延迟、成本看板。  
- **平台化能力**：SDK、模板、配置化，降低业务接入门槛。  
- **安全与合规**：敏感信息处理、权限隔离、日志留痕。

## 3. 常见面试题

### 基础题
- 如何设计一个统一的 LLM Gateway？  
- 平台层应提供哪些基础能力给业务团队？  
- 如何监控模型调用质量和成本？  
- 限流和熔断在 AI 平台里如何落地？

### 进阶题
- 多模型路由策略如何兼顾质量、成本和延迟？  
- 如何设计租户隔离与配额体系？  
- 如何发现并处理“成本异常上涨”问题？  
- 如何做跨团队可复用，而不是每个业务重复造轮子？

### 场景题
- 某模型厂商不稳定，如何实现快速切换与无感降级？  
- 双十一流量暴增，平台如何保 SLA？  
- 某租户请求异常占用大量预算，如何自动治理？

## 4. 面试回答要点

- 用“平台视角”回答：标准化、复用、治理。  
- 优先给治理手段：配额、路由、缓存、报警、兜底。  
- 用指标证明价值：接入效率、故障恢复时长、成本下降比例。

## 5. 面试题参考答案（精选）

### Q1：如何设计统一的 LLM Gateway？
**参考答案**：  
统一入口接收请求，做鉴权/限流/配额，然后根据策略路由到不同模型；返回统一响应格式并记录审计日志。这样业务无需关心底层模型差异。

### Q2：多模型路由如何兼顾质量、成本、时延？
**参考答案**：  
先定义分层策略：低复杂度请求走轻量模型，高复杂度或失败重试走高质量模型；配合离线评测和线上监控持续调阈值。

### Q3：成本异常上涨怎么处理？
**参考答案**：  
先定位是流量异常、路由失效还是 prompt 变长；短期先限额与降级止损，长期通过缓存、路由、提示词压缩和预算规则治理。

### Q4：厂商不稳定时如何无感降级？
**参考答案**：  
提前接入双 Provider 并做健康检查；主 Provider 异常自动切备；响应协议统一，业务层无感知。
