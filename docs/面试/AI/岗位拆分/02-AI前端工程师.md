# AI 前端工程师面试题与知识点

## 1. 岗位定位

- 核心目标：把 AI 能力做成“可用、好用、可解释”的交互产品。  
- 典型工作：对话 UI、流式渲染、引用展示、工具调用可视化、性能优化。

## 2. 知识点清单（讲清楚版）

- **前端基础与工程化**：React/Vue、状态管理、组件设计、性能优化。  
- **流式交互**：SSE/WebSocket、增量渲染、中断与重试。  
- **多轮会话管理**：上下文维护、历史折叠、会话恢复。  
- **可解释 UI**：引用来源、工具执行轨迹、置信提示。  
- **异常体验**：超时、失败、重试、降级、转人工。  
- **前后端协同**：接口约定、幂等、错误码、鉴权。  
- **基础安全**：XSS、内容过滤、敏感操作二次确认。

## 3. 常见面试题

### 基础题
- SSE 和 WebSocket 在 AI 对话中如何选择？  
- 如何实现逐字流式输出且避免页面卡顿？  
- 对话历史如何存储与恢复？  
- 如何在 UI 中展示“答案来自哪里”？

### 进阶题
- 大量消息场景如何做虚拟列表与增量更新？  
- 工具调用过程如何可视化，既透明又不干扰主流程？  
- 如何做前端级别的 token 成本提示与控制？  
- 长会话上下文变长导致响应慢，前端能做哪些优化？

### 场景题
- 设计一个“客服 AI 工作台”：对话区 + 工具面板 + 引用面板。  
- 用户频繁打断回答并追问，如何保证体验稳定？  
- 后端偶发超时，前端如何做到“感知好、不丢状态”？

## 4. 面试回答要点

- 重点体现“体验指标”：首字时间、完整响应时间、失败重试成功率。  
- 用真实交互细节证明能力：中断、撤回、重试、引用跳转。  
- 展示工程思维：可观测埋点、错误分层、性能预算。

## 5. 面试题参考答案（精选）

### Q1：SSE 和 WebSocket 在 AI 对话中如何选择？
**参考答案**：  
- 仅“服务端推送回答流”场景，优先 SSE，简单、稳定、成本低。  
- 需要双向实时交互（协同编辑、实时控制）才选 WebSocket。  
- 多数 AI 聊天产品先 SSE，后续复杂场景再补 WebSocket。

### Q2：如何实现逐字流式且不卡顿？
**参考答案**：  
不要每个 token 都触发整页渲染。做法是：token 先入缓冲区，按 30-60ms 批量 flush 到 UI；消息区用虚拟列表；重组件拆分 memo，避免整棵树频繁更新。

### Q3：如何在 UI 中展示“答案来自哪里”？
**参考答案**：  
每段回答关联引用片段（文档名、段落、位置）；点击可跳转原文。若无引用，则明确提示“本回答未命中知识库，仅为模型推断”，避免误导用户。

### Q4：后端偶发超时，前端如何做到不丢状态？
**参考答案**：  
消息状态机至少有 sending/streaming/success/failed；失败消息保留在会话中并支持“原地重试”；请求携带幂等 ID，避免重试后重复插入消息。
