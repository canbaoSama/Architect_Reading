# AI 面试准备路线与资源

## 一、准备时间规划（参考 3-5 个月）

| 阶段 | 周期 | 内容 |
|------|------|------|
| Python 基础 | 2-3 周 | 语法、环境、常用库，便于调用 API 和写脚本 |
| LLM API 使用 | 2-3 周 | OpenAI、Anthropic 等 API，流式响应、消息结构 |
| RAG 体系 | 3-4 周 | 分块、embedding、向量库、检索策略、评估 |
| **主流框架** | 2-3 周 | MCP、Rules/Skills、LangChain/LangGraph、OpenClaw（详见 [05-主流框架与工具栈](./05-主流框架与工具栈.md)） |
| 全栈 AI 项目 | 3-4 周 | 从产品到前端的完整 AI 应用，可展示可讲述 |

## 二、技能树（前端转 AI）

```
前端转 AI 技能树
├── 已具备（迁移）
│   ├── React/Vue 组件化思维 → AI 界面与状态管理
│   ├── 异步处理、流式数据 → 处理 AI 流式输出
│   └── 交互与体验 → AI 对话、加载、错误反馈
├── 需补充（核心）
│   ├── LLM 原理（Transformer、Tokenization、Embedding）
│   ├── 提示词工程（零-shot、few-shot、CoT）
│   ├── RAG 设计与实现
│   ├── FunctionCall / Agent 基础
│   └── Python（调用 API、处理数据）
└── 主流框架（2025-2026 重点）
    ├── OpenClaw：文件架构、Gateway、记忆与 RAG
    ├── LangChain / LangGraph：Chain vs Graph、状态管理、Agent 设计
    ├── Rules vs Skills：Cursor 定制、多步工作流
    ├── MCP：Tools/Prompts/Resources、与 AI 应用集成
    └── 向量数据库实战
```

## 三、简历与项目建议

### 简历重点

- 突出 **LLM 项目**，而非仅 ML 课程或传统前端
- 写明：RAG、Agent、流式输出、FunctionCall 等关键词
- 项目描述中体现：问题、方案、技术选型、效果

### 推荐项目方向

1. **RAG 知识库问答**：上传文档 → 检索 → 流式回答
2. **AI 聊天应用**：对话历史、流式输出、可选 FunctionCall
3. **Agent 小应用**：工具调用（天气、搜索、计算等）
4. **业务场景改造**：在现有系统中接入 AI（客服、文档、搜索等）

### 项目可展示的能力

- 需求拆解、方案设计
- RAG 或 Agent 架构理解
- 提示词设计与优化
- 流式 UI、错误处理、边界情况

## 五、面试常见流程

1. **Recruiter 沟通**（约 30 分钟）：岗位匹配、薪资、时间
2. **技术初筛**（45-60 分钟）：LLM 基础、简单 coding、API 调用
3. **技术深挖**（1-2 轮，各约 60 分钟）：RAG 设计、Agent、系统设计
4. **行为/文化**：过往项目、协作方式、解决问题思路

## 六、公司类型差异

| 类型 | GenAI 占比 |  coding | 系统设计 |
|------|------------|---------|----------|
| AI 初创 | 70-80% | 用 LLM API 现场实现 | 设计 AI 产品端到端 |
| 大厂 | 50-60% | LeetCode + API | 高并发、大规模 AI 系统 |
| 企业 | 60-70% | 常为 take-home AI 项目 | 与现有企业系统集成 |

---

**核心建议**：把「前端 + AI 界面 specialist」作为定位，既能做体验，又能理解 RAG、Agent 和 prompt，是当前较受欢迎的组合。
